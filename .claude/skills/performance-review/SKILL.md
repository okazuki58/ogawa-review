# 人事評価支援スキル（強化版）

## 概要

このスキルは、半期ごとの人事評価プロセスを効率化します。会社のMission/Value、グレード制度、評価基準に基づいて、目標設定から振り返りまでをサポートします。

**会社の半期定義**:
- **上半期（H1）**: 3月〜8月（6ヶ月）
- **下半期（H2）**: 9月〜2月（6ヶ月）

**v2の新機能**:
- 📊 階層的目標設定（半期→月次→週次→日次）
- 📁 macloggerログとの自動連携
- 🔄 実績の自動抽出・分析

## 使用タイミング

- 半期の目標設定を行う時
- 月次・週次・日次の目標に分解する時
- 期末の振り返り・自己評価を作成する時
- macloggerのログから実績を抽出する時
- 1on1の準備をする時
- 自分の実績が評価基準にどう該当するか確認したい時

## ログファイルの場所

このskillは、以下のログファイルを自動参照します：

### macloggerログ（自動収集済み）
- `../maclogger/reports/daily/*.md` - 日次ログ
- `../maclogger/reports/weekly/*.md` - 週次ログ

### プロジェクト内ログ（プロジェクトルートからの相対パス）
- プロジェクトルート/`logs/daily/*.md` - 日次ログ（シンボリックリンク推奨）
- プロジェクトルート/`logs/weekly/*.md` - 週次ログ（シンボリックリンク推奨）
- プロジェクトルート/`reviews/*.md` - 過去の振り返り
- プロジェクトルート/`work-log/*.md` - 手動作業ログ

**NOTE**: `logs/` ディレクトリがシンボリックリンクの場合、自動的にmacloggerのログを参照します。

## 主要機能

### 1. 階層的目標設定

#### 1-1. 半期目標の設定

ユーザーの現在のグレード、役割、過去の実績を踏まえて、以下の観点から**半期目標**を提案：

- **Be hungry**: 学習・成長・高みを目指す目標
- **Be an owner**: 当事者意識・責任感を示す目標
- **Be sustainable**: 継続性・効率化・健康を意識した目標

**プロセス**:
1. ユーザーの現状を深掘り（担当業務、技術スタック、課題感、キャリアイメージ）
2. グレード要件を確認
3. 3つのValueに沿った半期目標を提案
4. SMARTな目標になるよう調整

#### 1-2. 月次目標への分解

半期目標を達成可能な**月次目標**に分解：

```
半期目標: Railsのアーキテクチャパターンを学び、設計力を向上させる

↓ 分解

月次目標（6ヶ月分全部作成）:
- 1ヶ月目（9月）: 「Everyday Rails」を読み、基本パターンを理解
- 2ヶ月目（10月）: 学んだパターンをチームに共有、フィードバック収集
- 3ヶ月目（11月）: 小規模機能の設計ドキュメントを作成
- 4ヶ月目（12月）: 中規模機能の設計・実装
- 5ヶ月目（1月）: 設計レビューの実施
- 6ヶ月目（2月）: 半期の成果まとめ・発表
```

#### 1-3. 週次目標への分解

月次目標をさらに**週次目標**に分解（各月を4週に分解、計24週分）：

```
月次目標（9月）: 「Everyday Rails」を読み、基本パターンを理解

↓ 分解

週次目標:
- Week 1（9月第1週）: Chapter 1-3を読む
- Week 2（9月第2週）: Chapter 4-6を読む、サンプルコード写経
- Week 3（9月第3週）: Chapter 7-9を読む
- Week 4（9月第4週）: 学んだことをまとめ、勉強会資料作成

※同様に10月、11月、12月、1月、2月の各月も4週に分解（計24週）
```

#### 1-4. 日次目標への分解

週次目標をさらに**日次目標**に分解（直近1ヶ月分 = 4週 × 5日 = 20日分）：

```
週次目標（9月第1週）: Chapter 1-3を読む

↓ 分解

日次目標:
- 月曜: Chapter 1を読む（30分）
- 火曜: Chapter 2を読む（30分）
- 水曜: Chapter 3を読む（30分）
- 木曜: Chapter 1-3の復習、要点まとめ（30分）
- 金曜: 疑問点を調査・解決（30分）

※同様に9月第2週、第3週、第4週も日次に分解（計20日分）
※10月以降の日次目標は、各月開始時に改めて作成
```

### 2. ログからの実績抽出

macloggerの日次・週次ログから、自動的に実績を抽出・分析：

#### 実績抽出ワークフロー

```
1. ログファイルの検索
   - `../maclogger/reports/daily/` または `logs/daily/` から該当期間のログを取得

2. Value別に実績を分類
   - Be hungry: 学習、新技術導入、自己成長
   - Be an owner: 責任完遂、チーム貢献、問題解決
   - Be sustainable: 効率化、自動化、ドキュメント化

3. 具体的なエピソードを抽出
   - 日付、タスク内容、成果を構造化

4. 評価シートに自動記入
```

#### 実績抽出のコマンド例

```bash
# 過去6ヶ月のログから実績を抽出
> ../maclogger/reports/daily/ から過去6ヶ月の実績を抽出して、Value別に整理して

# 特定のValueに絞って抽出
> ログからBe sustainableな実績を抽出して

# 週次ログから月次サマリー作成
> ../maclogger/reports/weekly/ から今月の週次ログを集約して、月次レポートを作成して
```

### 3. 振り返り作成

以下のフォーマットで振り返りシートを自動生成：

1. **能力評価**: グレード別の役割・レベル感への到達度
2. **バリュー評価**: 各Valueに対する事実ベースの記載（ログから自動抽出）
3. **総合評価案**: 期待される評価ランクの予測

**ログ連携**:
- 半期の振り返り時、自動的に過去6ヶ月のログを分析
- 実績を時系列で整理
- Value別に分類して評価シートに記入

### 4. 評価基準の確認

- グレード昇進の可能性チェック
- 評価ランク（S/A/B/C/D）の予測
- 給与への影響の説明

## ワークフロー

### 目標設定時（階層化対応）

**CRITICAL**: 目標設定時は以下の順序で**必ず**実行すること

#### ステップ1: プロフィール情報の確認

```
1. my-profile が利用可能かチェック（プロジェクトルート/my-profile/）
   - 利用可能 → my-profile/SKILL.md を読み込み
   - references/*.md も読み込み（background, current-role, strengths-weaknesses, career-goals）
   
2. my-profile がない場合
   → ユーザーに profile-builder skill でプロフィール作成を促す
   「目標設定の前に、まずプロフィールを作成しましょう。
    profile-builder skill を使って、30分-1時間かけて高解像度のプロフィールを作成してください。
    プロフィールは目標設定の土台となる重要なステップです。」
   
3. プロフィール情報がある場合
   → ユーザーへの質問をスキップし、プロフィールの情報を使用
```

#### ステップ2: 評価基準の確認

```
1. references/grades.md でグレード要件を参照
2. references/values.md でValue基準を参照
3. 前回評価があれば プロジェクトルート/resources/original-sheets/配下のCSVを確認
```

#### ステップ3: 半期目標の提案

```
1. 3つのValueに沿った半期目標を提案
   - Be hungry: 学習・アウトプット
   - Be an owner: チーム貢献・発信
   - Be sustainable: 仕組み化・ドキュメント化

2. SMARTな目標になるよう調整
   - Specific（具体的）
   - Measurable（測定可能）
   - Achievable（達成可能）
   - Relevant（関連性）
   - Time-bound（期限）

3. 成果指標を明確に設定
```

#### ステップ4: 自動的に階層分解（MUST）

**重要**: 半期目標を提案したら、**自動的に**以下の分解を行うこと。ユーザーに「分解してください」と言われるのを待ってはいけない。

```
1. 月次目標に分解（半期分全部 = 6ヶ月分）
   - 各月の具体的なアクション
   - マイルストーン設定
   - 例: 2025H2なら9月、10月、11月、12月、1月、2月の6ヶ月分
   
2. 週次目標に分解（半期分全部 = 約24週分）
   - 各月を4-5週に分解
   - 各週の具体的なタスク
   - 週次の成果物
   - 例: 9月Week1-4、10月Week1-4、...、2月Week1-4
   
3. 日次目標に分解（直近1ヶ月分 = 約20日分）
   - 直近1ヶ月（4週）を日次に分解
   - 月〜金の具体的なタスク
   - 1日あたりの時間配分
   - 例: 9月第1週の月曜〜金曜、9月第2週の月曜〜金曜、...
   
4. ファイルに保存（プロジェクトルート/goals/配下に保存）
   - プロジェクトルート/goals/YYYY-HX-半期目標.md（1ファイル）
   - プロジェクトルート/goals/YYYY-HX-月次目標.md（1ファイルに6ヶ月分まとめて記載）
   - プロジェクトルート/goals/YYYY-MM-週次目標.md（各月ごとに1ファイル、計6ファイル）
   - プロジェクトルート/goals/YYYY-MM-WX-日次目標.md（直近1ヶ月の各週ごとに1ファイル、計4ファイル）
   
   **重要**: .claude/skills/performance-review/goals/ではなく、プロジェクトルートのgoals/に保存すること
```

**分解の粒度**:
- 月次: 半期分全部（6ヶ月）
- 週次: 半期分全部（24週）
- 日次: 直近1ヶ月分のみ（20日）
  ※日次は6ヶ月分（120日）作ると膨大になるため、直近1ヶ月のみ作成
  ※2ヶ月目以降の日次目標は、各月開始時に改めて作成

#### ステップ5: 確認と調整

```
1. 生成した目標をユーザーに提示
2. 調整が必要か確認
3. 必要に応じて修正
```

### 振り返り作成時（ログ連携対応）

```
1. 期間を確認（例: 2025H2 = 2025年9月〜2026年2月）
2. ../maclogger/reports/daily/ から該当期間のログを取得
3. references/templates.md のフォーマットを使用
4. ログから実績を自動抽出
   - Be hungry: 学習履歴、技術導入
   - Be an owner: 完遂タスク、チーム貢献
   - Be sustainable: 効率化、自動化
5. 能力評価：グレード基準との照合
6. バリュー評価：事実ベースで各項目を記載
7. 総合評価：S/A/B/C/Dの予測と根拠
8. プロジェクトルート/reviews/YYYY-HX.md に保存
```

### 壁打ち対話による深掘り

ユーザーが「壁打ちして深掘りしたい」と言った場合：

```
1. 現状の理解
   - 「現在の業務内容を詳しく教えてください」
   - 「具体的にどんなタスクをしていますか？」
   - 「最近完了したタスクは何ですか？」

2. 技術スタックの確認
   - 「主に使っている技術は？」
   - 「これから学びたい技術は？」
   - 「チームで使われている技術は？」

3. 課題の明確化
   - 「今、何が難しいですか？」
   - 「できるようになりたいことは？」
   - 「過去に失敗したことは？」

4. キャリアの方向性
   - 「次のグレードを目指していますか？」
   - 「マネージャーとスペシャリスト、どちらに興味がありますか？」
   - 「1年後、どうなっていたいですか？」

5. 具体的な目標の提案
   - 壁打ち内容を踏まえ、3つのValue観点から目標を提案
   - SMARTな目標になるよう調整
   - 月次・週次・日次に分解
```

## 重要な評価原則

1. **事実ベース**: すべての評価は具体的な事実・エピソードに基づく（ログから自動抽出可能）
2. **3つのValue**: Be hungry / Be an owner / Be sustainable の観点を常に意識
3. **グレード要件**: 現在のグレードの期待値を明確に理解する
4. **階層的目標**: 半期→月次→週次→日次に分解し、達成可能にする

## 評価ランクの基準

- **S (想定を大きく超えている)**: 給与アップ
- **A (期待値ちょい上)**: 給与アップ
- **B (期待値通り)**: 給与ステイ
- **C (期待値に達していない)**: 前回C以下の場合、給与ダウン
- **D (出来ているとはいえない)**: 給与ダウン

## 使用例

**ユーザー**: 「今期の目標を立てたい」

→ Claude: my-profile/ をチェック

    - my-profile/ を読み込み（現在のグレード、目標グレード、前回評価などを確認）
    - 現在のグレード要件を確認し、目標グレードを目指せる具体的な目標を提案
    - 半期→月次→週次→日次に分解
  
  - **my-profile/ がない場合**:
    - 「目標設定の前に、まずプロフィールを作成しましょう。
       profile-builder skill を使って、30分-1時間かけて高解像度のプロフィールを作成してください。

**ユーザー**: 「ログから今月のBe sustainable実績を抽出して」
→ 該当月のログを検索、効率化・自動化・ドキュメント化の実績をリストアップ

**ユーザー**: 「今週の目標の進捗を確認したい」
→ 週次目標と日次ログを照合、達成状況を可視化
