# プロフィールビルダー

## 概要

このskillは、**30分-1時間かけて作る選択式対話**でエンジニアの高解像度プロフィールを作成します。

プロフィールの設計は目標設定の土台となる最も重要なステップです。時間をかけて丁寧に作成します。

- **完全選択式**: 自由入力は最小限。具体的な選択肢から選ぶだけ
- **十分な深掘り**: 約40-50問、30分-1時間で完了
- **進捗表示**: 「Q1/50」など、常に進捗を表示
- **想定時間の提示**: 開始時に所要時間を明示
- **高解像度**: 選択式でも具体的で詳細なプロフィールを作成

## 使用タイミング

- 初回プロフィール作成時
- 半期ごとのプロフィール更新時
- グレード変更時
- 技術スタック変更時

## 選択肢自動生成の仕組み

1. `../performance-review/references/grades.md` からグレード情報を読み込み
2. `../../resources/original-sheets/` から前回評価を読み込み、B/C評価の項目を課題として提示
3. グレード・年収レンジから一般的なスキルレベルを推定
4. `../performance-review/references/values.md` から評価基準を読み込み、具体例を選択肢化
5. `references/grade-capabilities.md` から一般的な能力を読み込み
6. `references/tech-stacks.md` から技術スタック選択肢を読み込み

## プロフィール作成フロー

**所要時間**: 30分-1時間（約40-50問）
**形式**: 選択式中心（自由入力は最小限）

### 重要な原則

1. **必ず1つずつ質問**: 複数の質問を一度に投げかけない
2. **進捗を明示**: 「Q1/50」など、常に進捗を表示
3. **選択肢に必ず含める**:
   - 「全て当てはまる」（複数選択可の場合）
   - 「よくわからない」
   - 「その他（自由入力）」
4. **具体的な選択肢**: 抽象的な質問は避け、具体的な選択肢を提示
5. **前回評価は自動反映**: ユーザーに選ばせるのではなく、自動で読み込んで反映

## 出力ファイル生成

すべての質問に答えたら、`templates/profile-template.md`に基づいて以下のファイルを自動生成します。

### 生成ファイル

```
my-profile/
├── SKILL.md                      # メインプロフィール
└── references/
    ├── background.md             # 経歴・スキルセット
    ├── current-role.md           # 現在の業務・役割
    ├── strengths-weaknesses.md   # 強み・課題
    └── career-goals.md           # キャリア目標
```

### 生成ロジック

1. **選択式の回答を構造化**: 選択した項目をベースに文章化
2. **フォローアップ質問の回答を統合**: エピソード・感情・意欲を該当セクションに配置
3. **前回評価の課題を`strengths-weaknesses.md`に反映**: B/C評価項目とその背景を記載
4. **グレード・年収レンジから期待される能力を`background.md`に追記**: 一般的な能力レベルを補足
5. **技術スタックを構造化**: 使用年数・できること/できないことを明記
6. **キャリア目標を具体化**: 目指す方向性・達成イメージ・障壁と対策を記載

### 各ファイルへのマッピング

| ファイル | 含まれる情報 |
|---------|-------------|
| `SKILL.md` | 氏名、グレード、所属部署、エンジニア歴、メイン技術領域、キャリア方向性の概要 |
| `background.md` | 技術スタック（使用年数、できること/できないこと、印象的な経験、学びたい技術）、エンジニアとしてのスタート、成長の実感 |
| `current-role.md` | 開発スタイル、チームでの役割、担当範囲、各フェーズでの実績・苦手分野 |
| `strengths-weaknesses.md` | 強み（エピソード、周囲の評価、活かす場面）、課題（背景、苦労した場面、改善の試み） |
| `career-goals.md` | 目指す方向性（理由、意識し始めた時期、ロールモデル）、次の半期で達成したいこと（具体的なイメージ、障壁と対策） |

## 重要な設計原則

1. **十分な時間をかける**: 30分-1時間、全50問でしっかりプロフィールを作る
2. **完全選択式**: 自由入力は最小限。具体的な選択肢を提示
3. **一問一答形式**: 質問は必ず1つずつ。複数の質問を一度に投げかけない
4. **進捗を明示**: 「Q1/50」など、常に進捗を表示してユーザーの疲労を軽減
5. **選択肢の必須項目**:
   - 「全て当てはまる」（複数選択可の場合）
   - 「よくわからない」
   - 「その他（自由入力）」
6. **具体的な選択肢**: 抽象的な質問（「うまくいったことは？」）ではなく、具体的な選択肢を提示
7. **前回評価は自動反映**: ユーザーに選ばせるのではなく、CSVから自動で読み込んで反映
8. **既存情報を活用**: グレード・Value・前回評価から選択肢を自動生成
9. **段階的な質問**: 6ステップに分割（基本情報→技術→役割→強み課題→前回評価→キャリア）

## エラーハンドリング

## Claude Codeへの指示

このskillを実行する際は、以下を厳守してください：

### 必須事項

1. **開始時に所要時間を提示**: 「30分-1時間（全50問）」と明示
2. **必ず一問一答形式**: 質問は1つずつ。複数の質問を同時に投げかけない
3. **進捗を明示**: 「Q1/50」など、毎回進捗を表示
4. **ステップ表示**: 「【Step 1/6: 基本情報（8問）】」など、現在のステップを明示
5. **選択肢の必須項目**:
   - 「全て当てはまる」（複数選択可の場合）
   - 「よくわからない」
   - 「その他（自由入力）」
6. **選択肢は番号付きリスト**: 必ず番号をつける
7. **ユーザーの回答を待つ**: 回答がない状態で次の質問に進まない

### 既存プロフィールの確認方法

**重要**: `my-profile/`はプロジェクトルートに作成されます。`.claude/skills/`配下ではありません。

既存プロフィールを確認する際は：

```
プロジェクトルート/my-profile/SKILL.md
```

を探してください。`.claude/skills/my-profile/`ではありません。

### 前回評価の読み込み方法

**重要**: 前回評価はユーザーに「選ばせる」のではなく、CSVから自動で読み込みます。

1. `resources/original-sheets/`配下の最新CSVを読み込む
2. B/C評価の項目を自動抽出
3. Q37で「この課題について振り返ってください」と提示（選択肢: 改善できた/少し改善/まだ改善できていない）
4. ユーザーに「どの課題を重視するか選んでください」とは聞かない（課題は既に評価で決まっている）

### 抽象的な質問の禁止

以下のような抽象的な質問は**絶対に**しないでください：

❌ NG例:
- 「うまくいったことは何ですか？」
- 「できることと、まだできないことの境界線はどこですか？」
- 「苦労したことは何ですか？」

⭕ OK例:
- 「以下の経験はありましたか？（複数選択可）」+ 具体的な選択肢
- 「できることを選んでください（複数選択可）」+ 具体的な選択肢
- 「苦労した場面を選んでください（複数選択可）」+ 具体的な選択肢
